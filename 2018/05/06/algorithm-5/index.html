最短路径--Bellman-Ford 解决负权边

By zzulixjh

Published 2018-05-06

Contents

Dijkstra算法虽然好，但是它不能解决带有负权边(边的权值为负数）的图。本节我要来介绍一个无论是思想上还是代码上都堪称完美的最短路算法:Bellman-Ford。Bellman-Ford算法非常简单，核心代码只有4行，并且可以完美地解决带有负权边的图，先来看看它长啥样:

上面的代码中，外循环一共循环了n-1次(n为顶点的个数)，内循环循环了m次(m为边的个数)，即枚举每一条边。dis数组的作用与Dijkstra算法一样，是用来记录源点到其余各个顶点的最短路径。u\v和w三个数组是用来记录边的信息。例如第i条边存储在u[i]、v[i]、和w[i]中，表示从顶点u[i]到顶点v[i]这条边(u[i]->v[i])权值为w[i]。
if(dis[v[i]]>dis[u[i]]+w[i])
dis[v[i]] = dis[u[i]] + w[i];
上面这两行代码的意思是:看看能否通过u[i]->vi这条边，使得1号顶点到v[i]号顶点的距离变短。即1号顶点到u[i]号顶点的距离(dis[u[i]])加上u[i]->v[i]这条边（权值为w[i]）的值是否会比原先1号顶点到v[i]号顶点的距离(dis[v[i]])要小。这一点其实是与DIJKSTRAD “松弛”操作是一样的。现在我们要把所有的边都松弛一遍，代码如下。

下面举个例子，求下图中1号顶点到其余所有顶点的最短路径。



第一轮在对所有边进行松弛之后，得到的是从1号顶点”只能经过一条边”到达其余各个顶点的最短路径长度。第2轮在对所有的边 进行松弛之后，得到的是从1号顶点”最多经过两条边”到达其余各个顶点的最短路径长度。如果进行k轮的话，得到的就是1号顶点”最多经过k条边”到达其余各顶点的最短路径长度，那么需要进行多少轮呢?答案是n-1轮。因为在一个含有n个顶点的图中，任意两点之间的最短路径最多包含n-1边。

我们可以看到第4轮之后dis数组没有发生任何变化，所有说是最多进行n-1轮松弛。
整个Bellman-Ford算法用一句话概括就是:对所有的边进行n-1次”松弛”操作。核心代码只有4行，如下。

完整代码

#include
int main()
{
int dis[10],i,k,n,m,u[10],v[10],w[10];
int inf=99999999;
//读入n和m分别表示n个顶点和m条边
scanf(“%d%d”,&n,&m);
//读入边
for(i=1;i<=m;i++)
{
scanf(“%d%d%d”,&u[i],&v[i],&w[i]);
}
for(i=1;i<=n;i++)
{
dis[i]=inf;
}
dis[1]=0;
//Bellman-Ford算法核心语句
for(k=1;k<=n-1;k++)
{
for(i=1;i<=m;i++)
{
if(dis[v[i]] > dis[u[i]] + w[i] )
{
dis[v[i]] = dis[u[i]] + w[i];
}
}
}
//输出最终的结果
for(i=1;i<=n;i++)
{
printf(“%d “,dis[i]);
}
return 0;
}
我们还可以对这个算法进行优化，来减少它的复杂度。
