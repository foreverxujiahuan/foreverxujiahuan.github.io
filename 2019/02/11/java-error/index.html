什么是违例


违例这个词表达的是一种”例外”的情况,亦即正常情况之外的一种”异常”

违例自变量


和Java的其他任何对象一样，需要用new在内存堆里创建违例，并需调用一个构建器。在所有标准违例中，存在着两个构建器:第一个是默认构建器，第二个则需要使用一个字串自变量，使我们能在违例里置入相关信息,例如:
throw new NullPointerExeption(“t=null”)
违例的抛出可以终止方法或作用域的继续。

违例准则


用违例做下面这些事情:
1.解决问题并再次调用造成违例的方法。
2.平息事态的发展，并在不重写尝试方法的前提下继续。
3.计算另一些结果，而不是希望方法产生的结果。
4.在当前环境中尽可能解决问题，以及将相同的违例重新”掷”出一个更高级的环境。
5.在当前环境中尽可能解决问题，以及将不同的违例重写”掷”出一个更高级的环境。
6.终止程序执行。
7.简化编码。若违例方案使事情变得更加复杂，那就会令人非常烦恼，不如不用
8.使自己的库和程序变得更加安全。这既是一种”短期投资”(便于调试)，也是一种”长期投资”(改善应用程序的健壮性)

违例控制器


生成的违例必须在某个地方终止。这个”地方”便是违例控制器或者违例控制模块。而且针对想捕获的每种违例类型，都必须有一个相应的违例控制器。违例控制器紧接在try块后面，且用catch(捕获)关键字标记。

控制器必须紧接在try块后面。若”掷”出一个违例，违例控制机就会搜寻自变量与违例类型相符的第一个控制器。随后，它会进入那个catch从句，并认为违例已得到控制。

可以看到在getMessage()方法可以获得违例中的违例信息，toSring()可以获取抛出的所有违例信息

总结


通过先进的错误纠正与恢复机制，我们可以有效地增强代码的健壮度。对我们编写的每个程序来说，错误恢复都属于一个基本的考虑目标。它在Java中先得尤为重要，因为该语言的一个目标就是创建不同的程序组件，以便其他用户(客户程序员)使用。为构建一套健壮的系统，每个组建都必须非常健壮。
在Java里，违例控制的母的是使用尽可能精简的代码创建大型、可靠的应用程序，同时派出程序里那些不能控制的错误。
